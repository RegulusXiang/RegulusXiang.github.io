    A type is a set of values

    A subtype T2 of T1 is a subsets of the values of T1

    Subtype Polymorphism

    A function can be applied to an object (value) of any type that is a subtype of the specified type.

    • includes the specified types

    Is there a subtype relationship between function types?

    T1 $\to$ T2

    T3 $\to$ T4

    Can there be a subtype relationship?

    • yes

    Example. Is there a relationship between A $\to$ int and B $\to$ int

    Not safe to treat at B -> int as if it were an A -> int

    • so B -> int is not a subtype of A -> int

    Any where a B -> int is expected an A -> int can be used.

    • so A -> int is a subtype of B -> int

    Subtyping of functions is contrarients to the input type.

    If T2 is the subtype of T1, then T1 -> T for any T is the subtype of T2 -> T for any T.

    “B -> int”: the set of all functions that can be applied to a B object;

    “A -> int”: the set pf all functions that can be applied to a A object.

    Any function that expects an A can also be applied to B.

    A -> int $\subseteq$ B -> int

    There are functions in B -> int that cannot be applied to an A

    B -> int not subset of A -> int

    What is the subtyping relationship between int -> A and int -> B?

    Anywhere I’m expecting an int -> A can use an int -> B

    Vice not versa.

    But, if I’m expecting an int -> B, I can’t use an int -> A.

    int -> B is a subtype of int -> A

    The set of all functions that returns a A includes the set of all functions that return a B, for a B is an A.

    Function subtyping is covariant on the output types

    • If T2 is a subtype of T1, then T -> T2 is a subtype of T -> T1, for any T

    Function Subtyping

    If T1 is a subtype of T3, and T4 is a subtype of T2, then T3 -> T4 is a subtype of T1 -> T2.

    Java Generics

    Type check: Compiling time, not run time (eraser)

    In Java, there is no subtyping relationship between instances and generic classes.
